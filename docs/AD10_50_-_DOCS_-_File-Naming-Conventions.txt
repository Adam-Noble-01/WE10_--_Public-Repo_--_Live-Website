# NOBLE ARCHITECTURE FILE NAMING CONVENTIONS
--------------------------------------------------------------------------------

## DOCUMENT OVERVIEW

#### Document Purpose
This document establishes file naming conventions for the Noble Architecture website project.

#### Document Introduction Section
- Consistent file naming is essential for project organization
- These conventions apply to all files in the project
- Following these conventions ensures proper project structure

--------------------------------------------------------------------------------

## NAMING CONVENTIONS

### GENERAL RULES
--------------------------------------------------------------------------------
- Use lowercase for all file names
- Use hyphens (-) to separate words
- Avoid spaces and special characters
- Keep names descriptive but concise
- Include file type indicators where appropriate

### FILE TYPE SPECIFIC CONVENTIONS
--------------------------------------------------------------------------------
#### JavaScript Files
- Use .js extension
- Prefix with purpose (e.g., util-, core-, app-)
- Example: core-web-utility.js

#### CSS Files
- Use .css extension
- Group by component or feature
- Example: main-styles.css

#### HTML Files
- Use .html extension
- Name matches page purpose
- Example: about-page.html

### ASSET NAMING
--------------------------------------------------------------------------------
#### Images
- Use descriptive names
- Include dimensions where relevant
- Example: hero-banner-1920x1080.jpg

#### Documents
- Use prefix for document type
- Include version numbers where needed
- Example: AD10_40_-_DOCS_-_Coding-Standards.txt
